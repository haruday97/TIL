# Spring Boot Introduction
Spring Boot는 엔터프라이즈 애플리케이션을 개발하기 위한 핵심 기능을 제공하는 Spring Project 중 하나이다.  
Spring Framework의 편리성에도 불구하고 Spring 설정의 어려움을 겪는 문제점을 해결하기 위해 등장한 스프링 프로젝트이다.  

## Spring Boot를 사용해야하는 이유
<ul>
    <li>
    XML 기반의 복잡한 설계 방식 지양 : 스프링 부트는 XML 설정 파일을 대신하여 자바 어노테이션과 자바 코드를 이용한 설정 방식을 선호한다. 따라서 더 간결하고 가독성 높은 코드를 작성할 수 있다.
    </li>
    <li>
    의존 라이브러리의 자동 관리 : 스프링 부트 이전에는 애플리케이션에서 필요한 라이브러리를 사용하기 위해 필요한 라이브러리 이름과 버전을 일일이 추가해주어야 했다. 스프링 부트에서는 라이브러리를 자동으로 관리해주기에 개발자의 수고를 덜어준다.
    </li>
    <li>
    애플리케이션 설정의 자동 구성 : 스프링 부트는 개발자가 별도로 설정하지 않아도 애플리케이션의 기본 설정을 자동으로 구성한다.
    </li>
    <li>
    프로덕션급 애플리케이션의 쉬운 빌드 : 스프링 부트는 Maven, Gradle 등의 빌드 도구와 통합되어 있으며 손쉽게 애플리케이션을 빌드할 수 있다. 빌드 과정에서 애플리케이션을 패키징하고 의존 라이브러리를 포함하는 실행 가능한 JAR파일을 생성할 수 있다.
    </li>
    <li>
    내장된 WAS를 통한 쉬운 배포 : 스프링 부트는 내장된 Tomcat, Jetty, Undertow같은 WAS를 제공하며 이를 통해 개발자는 별도의 WAS 설치나 구성없이 애플리케이션을 실행할 수 있다.
    </li>
</ul>

즉 스프링을 사용하는 이유를 다음과 같이 요약할 수 있다.

> _스프링 구성은 스프링에 맡겨버리고 비즈니스 로직에만 집중하자!_
